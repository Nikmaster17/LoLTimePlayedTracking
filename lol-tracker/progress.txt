=============================================================
KNOWN ROUGH EDGES (AS OBSERVED IN CODE)
=============================================================

  1. No mobile-responsive layout — zero breakpoint prefixes used.
     The 3-col stat grid and 3-col form row do not reflow on small screens.

  2. Stats section hidden when user has zero sessions — no onboarding prompt
     or empty-state call-to-action.

  3. Only today's sessions shown on dashboard — no way to view/edit
     sessions from other dates without going to the calendar (which is read-only).

  4. Calendar is read-only — users who realize they missed logging yesterday
     must go back to dashboard, change the date picker, and log from there.
     The workflow is not obvious.

  5. No delete confirmation dialog — sessions deleted immediately on click.

  6. formatTime(0) returns "0m" everywhere (via lib/utils.js) — calendar cells
     with 0 minutes show no time sub-label (guarded by `mins > 0` check),
     which is correct, but the legend label "None" is the only indicator.

  7. Stats cards are a 3×2 grid but the 4th stat (Streak) appears in position
     4 (2nd row, 1st column), which may look unbalanced with the "Play today!"
     warning text expanding that cell's height.

  8. Fetching all sessions every time (no pagination, no date range filter) —
     acceptable now but will slow as session count grows.

  9. No visual loading skeleton — just a plain text "Loading..." string.

  10. Inline-edit inputs use `rounded` (not `rounded-lg`) — visual inconsistency
      with the main form inputs.
